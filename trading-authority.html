Usage

In a browser (ESM)

Place the script and import it with a relative path.

<script type="module">
import { 
  getTypedDataForSignature, 
  generateTradingAuthority
} from './trading-authority.js';

// 1) Prepare EIP-712 typed data
const domain = { name: 'authority', version: 'v1' };
const chainId = 11155111; // e.g. Sepolia
const nonce = 0; // Increase per new authority
const typed = getTypedDataForSignature({ domain, chainId, nonce });

// 2) Ask the wallet to sign
const signature = await walletClient.signTypedData({ account, ...typed });

// 3) Derive the Trading Authority key
const { mnemonic, privateKey } = await generateTradingAuthority(
  walletClient,
  { account, domain, chainId, nonce }
);
console.log(privateKey);
</script>

API

getTypedDataForSignature — Builds the EIP‑712 payload for onboarding. Provide the app name and version, a chain id, and a nonce. The result is an immutable object that can be passed directly to a wallet’s typed‑data signing method.

generateTradingAuthority — One‑call helper. Requests a signature from the connected wallet and deterministically derives the Trading Authority. Returns a mnemonic and a hex private key. The call fails if the signature is rejected or inputs are invalid.

deriveHDKeyFromEthereumSignature, entropyFromSignature, exportMnemonicAndPrivateKey — Lower‑level building blocks. Derive straight from a 65‑byte signature, extract a 32‑byte entropy buffer from it, or derive from a raw 32‑byte entropy source using the standard path m/44'/60'/0'/0/0. Handy for custom UX and testing.

ETH_DERIVATION_PATH, DEFAULT_PRIMARY_TYPE, DEFAULT_MESSAGE_TEMPLATE — Constants for the default derivation path, EIP‑712 primary type, and default action/notice strings.

Tip: for high-level implementations, always use the building blocks for full control; the one-call helper is available for convenience but is not the default approach.
